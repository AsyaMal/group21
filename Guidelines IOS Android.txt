Guidelines IOS Android

1. Guidelines 
IOS: Human Interface Guidelines c 2013
Android: Material design c 2014

2. Четких правил и конкретные спецификации
IOS: нет
Android: есть

3. Единицы измерения
IOS: pt
Android: dp

4. Размер экрана.Наименьший размер экрана под который проектируютмя приложения
IOS: 320pt x 568pt
Android: 360dp x 640dp

5. Системный шрифт
IOS: San Francisco
Android: Roboto

6. Встроенный инструмент навигации назад
IOS: нет
Android: Android Navigation Bar

7. Тени
IOS: нет
Android: есть. Есть ось Z

8. Инструмент предупреждения пользователя
IOS: Один - Alerts
Android: Три - Snackbars, Banners и Dialogs. Snackbar предназначен для сообщений низкой приоритетности и не требует действий. Dialogs блокирует взаимодействие с интерфейсом и требует совершить действие. Banners находятся между ними: не блокирует взаимодействие, но требует совершить действие.

9. Верхнеуровневая навигация
IOS: Tab bar
Android: Navigation Drawer, Bottom Navigation Bar и Tabs.

10. Разное поведение поиска
IOS: поиск относится к bar
Android: поиск - это еще один способ навигации
Both:  поиск может статично присутствовать на экране и, как правило, прибит к Navigation Bar/Top App Bar.

11. Разный вид стрелки назад 
IOS: Cтрелка не имеет линии посередине
Android: Cтрелка имеет линию посередине

12. Разный вид иконки трех точек
IOS: Горизонтальное положение
Android: Вертикальное положение

13. Разный вид Picker-а
IOS: выбор даты происходит с помощью барабана. Барабан iOS можно использовать для ввода любых других данных.
Android: Picker даты имитирует вид физического календаря.

14. Разные Text Fields
IOS:Label находится внутри поля ввода и исчезает во время ввода текста.
Android: Material рекомендует поднимать Label при вводе текста. Material также рекомендует выделять Label и полосу под Text Field основным цветом — это помогает понять, что поле выделено. Material описывает поведение поля при ошибке ввода. В Material на выбор есть две формы: Filled и Outlined.
Both: Обе платформы советуют при необходимости добавлять Clear Button.

15. Edit Menus vs Text Selection Tool Bar
IOS: Edit Menus. В iOS после долго нажатия возникает лупа для точного выбора места в слове.
Android: Text Selection Tool Bar. При долгом нажатии в Android пользователь может продолжить выделение текста. при вызове дополнительных действий Text Selection Tool Bar принимает форму Menus.

16. Разный размер divider-а
IOS: 0,5pt
Android: 1dp

17. Разные требования к размеру зоны нажатия
IOS: 44x44pt
Android: 48x48dp

18. Магазин приложений
IOS: App Store
Android: Google Play

19. Отношение к Branded Launch
IOS: Не рекомендует использовать Launch Screen для маркетинговых целей, и предлагает при запуске приложения выводить только Placeholder.
Android: Разрешает

20. Контекстные действия
IOS: Context Menus. контрол предлагает пользователю нескольких контекстных действий, связанных с выбранным элементом.
Android: Menus андроида применяется в большем числе кейсов: он предлагает контекстные действия как для выбранного элемента, так и для всей страницы в целом; используется как инпут с несколькими вариантами на выбор (дропдаун меню); применяется для редактирования текста.
